# Security Policy

FilDOS is a desktop application with full access to your filesystem, so security
reports are taken seriously.

## Reporting a vulnerability

Please do not open a public issue for security vulnerabilities.

Report privately through GitHub's
[security advisories](https://github.com/ahmedfahim21/FilDOS/security/advisories/new),
or by email to **ahmed.fahim0207@gmail.com**.

Include enough detail to reproduce the issue: the affected version or commit,
your operating system, the steps to trigger it, and the impact you observed. A
proof of concept is helpful but not required.

You can expect an acknowledgment within a few days. We will keep you updated on
progress and let you know when a fix ships. Please give us a reasonable window to
address the issue before any public disclosure.

## Supported versions

FilDOS is pre-1.0 and under active development. Fixes are applied to the `main`
branch; please make sure you can reproduce an issue against the latest `main`
before reporting.
