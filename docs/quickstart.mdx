FilDOS is pre-1.0 and doesn't ship packaged installers yet, so running it today
means building from source. It's a single `npm install` away.

<Note>
  Once signed, packaged builds are published they'll appear on the
  [Releases page](https://github.com/ahmedfahim21/FilDOS/releases).
</Note>

## Prerequisites

<Steps>
  <Step title="Node.js 22 or newer">
    Required — FilDOS relies on the built-in `node:sqlite` module, which needs
    Node 22 / Electron ≥ 35.
  </Step>
  <Step title="npm">
    Ships with Node. Used for install and all project scripts.
  </Step>
</Steps>

## Run the desktop app

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/ahmedfahim21/FilDOS.git
    cd FilDOS
    ```
  </Step>
  <Step title="Install dependencies">
    This also installs the git pre-commit hook.

    ```bash
    npm install
    ```
  </Step>
  <Step title="Start the dev server">
    ```bash
    npm run dev
    ```

    electron-vite starts with HMR for the renderer; the main and preload
    processes rebuild on change.
  </Step>
</Steps>

## Build a production bundle

```bash
npm run build    # output in out/
npm start        # preview the production build
```

## Optional: connect a cloud drive

Cloud-provider OAuth credentials (Google Drive, Dropbox) are optional and only
needed if you want to test those integrations locally. Copy `.env.example` to
`.env` and fill in the values. See [Cloud Drives](/features/cloud-drives) for
details.

## Verify your setup

Run the checks to confirm everything is wired up correctly:

```bash
npm test            # unit + integration (Vitest)
npm run typecheck   # TypeScript (both main and renderer projects)
npm run lint        # ESLint flat config
```

All three should exit with no errors on a clean clone.
