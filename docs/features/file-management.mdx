## Views

Switch between three layouts from the **View** menu:

- **List** ‚Äî a virtualized table with resizable, reorderable columns.
- **Grid** ‚Äî thumbnails at a few icon sizes, with cached previews.
- **Gallery** ‚Äî a larger-preview layout for browsing images.

Your choice is remembered per folder. A deliberate change to sort order, view
mode, or icon size becomes the new global default *and* the current folder's
remembered view ‚Äî navigate away and back and the folder looks the way you left
it.

## Undo

Every mutation ‚Äî copy, move, rename, duplicate ‚Äî pushes an inverse operation onto
the undo stack. `‚åò Z` replays it.

<Note>
  Delete is intentionally excluded from undo. `‚åò ‚å´` moves items to the **OS
  Trash / Recycle Bin** via the system API; recover them from Finder or Explorer
  if needed. FilDOS keeps no trash of its own.
</Note>

## Live updates

The file list subscribes to a non-recursive watch on the current directory.
Changes made by other apps (Terminal, Finder, an installer) appear in under a
second, debounced to avoid redundant reloads.

## Drag and drop

- **Internal** (within FilDOS): move by default, hold `‚å•` / `Alt` to copy.
- **Drag out** to Finder or Explorer: copies the file(s) to the drop target.
- **Drag in** from Finder or Explorer: moves or copies into the current
  directory.

## Recents

FilDOS keeps a persistent **Recents** list that survives across sessions and
updates automatically as files move. Open it from the sidebar. Like tags, recents
are stored in SQLite and follow files through renames and moves ‚Äî see
[Tags & Recents](/features/tags).
